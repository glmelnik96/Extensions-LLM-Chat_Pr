# Известные ошибки и непротестированные области

Документ без правок кода: фиксирует наблюдения и пробелы тестирования. При исправлении в коде — обновить или удалить соответствующий пункт.

---

## Известные ошибки / ограничения (подтверждены использованием или репортом)

### 1. Транскрибация: сообщение «Файл слишком большой; задайте exportAudioPresetPath + .epr.»

- **Когда:** путь `media_file` (один клип с `getMediaPath`), размер файла на диске превышает `maxTranscribeUploadBytes` в `fm-defaults.js`, либо нет рабочего экспорта In–Out через `.epr` и модель уходит в слишком большой одиночный файл.
- **Смысл для пользователя:** для выбранного сценария нужен пресет только аудио (`.epr`) и/или меньшие чанки/лимиты; см. `host/presets/README.txt` и поля `exportAudioPresetPath`, `transcribeExportChunkSec`, `maxTranscribeUploadBytes`.
- **Статус:** ожидаемое поведение проверки размера; UX может путать, если пресет уже есть, но сработал другой режим подготовки.

### 2. Кнопка «Стоп» не останавливает текущую операцию

- **Наблюдение:** прерывание рассчитано на **ожидание HTTP** к FM и шаги цикла агента (`abortCheck` / `signal`). Выполняющийся **синхронный `evalScript`** в Premiere (длинный экспорт, тяжёлая нарезка) **нельзя прервать** из панели до возврата в JS.
- **Статус:** ограничение CEP + ExtendScript; «Стоп» не гарантирует мгновенную остановку хоста.

### 3. Обрезание / удаление интервала → «бесконечные дубликаты» (агент)

- **Наблюдение:** при запросах через **чат (LLM)** модель может зациклиться: повторные `tool_calls`, попытки «убрать дубликаты» после split, лишние шаги без нового снимка.
- **Контекст:** после `ripple_delete_range` два фрагмента одного исходника на таймлайне — **нормальный split**, не обязательно ошибка; часть сценариев смягчена промптом и быстрым путём без LLM на панели таймкодов.
- **Статус:** риск на стороне оркестрации LLM, не детерминированный баг одной функции хоста.

### 4. Зацикливание агента — модель повторяет одинаковые tool_calls

- **Когда:** сложные задачи редактирования, при которых модель вызывает один и тот же инструмент с одними и теми же аргументами 3+ раз подряд.
- **Корневая причина:** open-source модели (gpt-oss-120b, GLM) слабее GPT-4/Claude в function calling.
- **Статус:** ✓ **ИСПРАВЛЕНО.** `agent-loop.js`: `hashToolCall()` + `detectCycle()` — force-stop после 3 идентичных вызовов. Покрыто юнит-тестами.

### 5. Нет стриминга — UI замирает во время ответа LLM

- **Статус:** ✓ **ИСПРАВЛЕНО.** `cloudru-client.js`: `parseSSEStream()` — SSE-парсер с агрегацией tool_calls deltas. `panel.js` передаёт `stream: true`.

### 6. Последовательные tool_calls — лишняя задержка

- **Статус:** ✓ **ИСПРАВЛЕНО.** `agent-loop.js`: `Promise.all` для параллельного выполнения tool_calls.

### 7. Нет повторных попыток при транзиентных ошибках API

- **Статус:** ✓ **ИСПРАВЛЕНО.** `cloudru-client.js`: `fetchWithRetry()` — 3 попытки с exponential backoff (1с/2с/4с), только для 5xx/429.

### 8. Невалидный JSON в аргументах tool_call

- **Статус:** ✓ **ИСПРАВЛЕНО.** `agent-loop.js`: `repairJson()` + `safeParseArgs()` — ремонт trailing commas, unquoted keys, обрезанных строк. Покрыто тестами.

### 9. Ложные срабатывания классификации намерений

- **Когда:** «удали маркеры» триггерит и TIMELINE, и MARKERS tiers. «Собери highlight reel» не совпадает ни с чем → загружаются ВСЕ tiers.
- **Влияние:** лишние prompt-токены, потенциальная путаница модели.
- **Статус:** regex-based `classifyIntent` в `prompts.js` — работает для типичных случаев, но имеет edge cases.

## Непротестированные или слабо покрытые функции

Автотесты в репозитории покрывают **только** `tool-validators.js` (аргументы планов). **Нет** интеграционных тестов против живого Premiere.

| Область | Комментарий |
|---------|-------------|
| `host/premiere.jsx` целиком | Снимок, `applyTimecodeEdits`, `applyTranscriptCuts`, маркеры, `prepareTranscribeFromTimeline`, `undoLast`, `ripple_delete` / split — только ручная проверка в Premiere; сценарии не хранятся отдельным чек-листом в репозитории. |
| Транскрибация: `export_chunks`, `clip_queue` | Очередь чанков и несколько клипов — без формального регрессионного плана в репо. |
| Whisper / FM | Таймауты, 413, не-JSON ответы, лимиты прокси — зависят от окружения и квот. |
| Панель таймкодов: быстрый путь по фразе | Регексы для «удали между N и M сек» — не покрыты тестами. |
| `undoLast` | Зависит от локализации меню и фокуса; не автотестируется. |
| Связка V+A на разных дорожках | После вырезаний возможен рассинхрон; нет формального тест-плана. |
| Кнопка «Стоп» во время `fetch` vs во время `evalScript` | Поведение различается; полная матрица не задокументирована тестами. |
| Детекция зацикливания агента (повторные tool_calls) | ✓ Реализовано и покрыто юнит-тестами (`_agentLoopInternals`). |
| Стриминг SSE (парсинг) | ✓ Реализовано (`parseSSEStream`). Не покрыто юнит-тестами — требует mock-сервера. |
| Параллельное выполнение tool_calls | ✓ Реализовано (`Promise.all`). Не покрыто юнит-тестами. |
| JSON repair | ✓ Реализовано (`repairJson`, `safeParseArgs`). Покрыто юнит-тестами. |
| Retry-логика | ✓ Реализовано (`fetchWithRetry`). Не покрыто юнит-тестами — требует mock HTTP. |
| J-cut / L-cut операции | ✓ Реализовано (`applyJCuts` в ExtendScript + bridge + UI). Ручная проверка в Premiere. |
| Детерминированные пайплайны (slash-commands) | ✓ Реализовано (`deterministic-pipelines.js`). Покрыто юнит-тестами. |
| Двухмодельная маршрутизация | ✓ Реализовано (`classifyComplexity`). Покрыто юнит-тестами. |
| Snapshot caching (dirty-flag) | ✓ Реализовано. Ручная проверка в Premiere. |
| Edge cases классификации намерений | Regex `classifyIntent` не покрыт юнит-тестами. |
