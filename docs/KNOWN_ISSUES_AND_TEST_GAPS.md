# Известные ошибки и непротестированные области

Документ без правок кода: фиксирует наблюдения и пробелы тестирования. При исправлении в коде — обновить или удалить соответствующий пункт.

---

## Известные ошибки / ограничения (подтверждены использованием или репортом)

### 1. Транскрибация: сообщение «Файл слишком большой; задайте exportAudioPresetPath + .epr.»

- **Когда:** путь `media_file` (один клип с `getMediaPath`), размер файла на диске превышает `maxTranscribeUploadBytes` в `fm-defaults.js`, либо нет рабочего экспорта In–Out через `.epr` и модель уходит в слишком большой одиночный файл.
- **Смысл для пользователя:** для выбранного сценария нужен пресет только аудио (`.epr`) и/или меньшие чанки/лимиты; см. `host/presets/README.txt` и поля `exportAudioPresetPath`, `transcribeExportChunkSec`, `maxTranscribeUploadBytes`.
- **Статус:** ожидаемое поведение проверки размера; UX может путать, если пресет уже есть, но сработал другой режим подготовки.

### 2. Кнопка «Стоп» не останавливает текущую операцию

- **Наблюдение:** прерывание рассчитано на **ожидание HTTP** к FM и шаги цикла агента (`abortCheck` / `signal`). Выполняющийся **синхронный `evalScript`** в Premiere (длинный экспорт, тяжёлая нарезка) **нельзя прервать** из панели до возврата в JS.
- **Статус:** ограничение CEP + ExtendScript; «Стоп» не гарантирует мгновенную остановку хоста.

### 3. Обрезание / удаление интервала → «бесконечные дубликаты» (агент)

- **Наблюдение:** при запросах через **чат (LLM)** модель может зациклиться: повторные `tool_calls`, попытки «убрать дубликаты» после split, лишние шаги без нового снимка.
- **Контекст:** после `ripple_delete_range` два фрагмента одного исходника на таймлайне — **нормальный split**, не обязательно ошибка; часть сценариев смягчена промптом и быстрым путём без LLM на панели таймкодов.
- **Статус:** риск на стороне оркестрации LLM, не детерминированный баг одной функции хоста.

## Непротестированные или слабо покрытые функции

Автотесты в репозитории покрывают **только** `tool-validators.js` (аргументы планов). **Нет** интеграционных тестов против живого Premiere.

| Область | Комментарий |
|---------|-------------|
| `host/premiere.jsx` целиком | Снимок, `applyTimecodeEdits`, `applyTranscriptCuts`, маркеры, `prepareTranscribeFromTimeline`, `undoLast`, `ripple_delete` / split — ручная проверка на разных секвенциях и версиях PP. |
| Транскрибация: `export_chunks`, `clip_queue` | Очередь чанков и несколько клипов — нет регрессионного чек-листа в репо. |
| Whisper / FM | Таймауты, 413, не-JSON ответы, лимиты прокси — зависят от окружения и квот. |
| Панель таймкодов: быстрый путь по фразе | Регексы для «удали между N и M сек» — не покрыты тестами. |
| `undoLast` | Зависит от локализации меню и фокуса; не автотестируется. |
| Связка V+A на разных дорожках | После вырезаний возможен рассинхрон; нет формального тест-плана. |
| Кнопка «Стоп» во время `fetch` vs во время `evalScript` | Поведение различается; полная матрица не задокументирована тестами. |
