## Часть 1 — Умные переносы субтитров (гибрид: детерминизм + LLM-подсказка)

### Проблема
`_wrapWords` (reels-pipeline.js:33) — чисто жадный перенос по символам, без
учёта смысла: предлог/союз может оказаться последним в строке («висячий
предлог»). LLM-корректор (`reelsProofread`, panel.js:7673) правит только
опечатки, а переносы после корректуры пересобираются по старой построчной
разбивке (`applyProofread`:244). Смысловые переносы не лечатся.

### Дизайн (гибрид)
Детерминированный слой ВСЕГДА гарантирует лимиты + правило «висячего предлога» +
балансировку строк. LLM даёт необязательную подсказку границы фразы (`\n`),
которую детерминированный слой валидирует и применяет только если она корректна.
Guard содержимого (число/порядок слов) — без изменений.

### Задачи
**T1.1 — `wrapCueLines` в reels-pipeline.js (TDD, ES5).**
Новая функция `wrapCueLines(words, maxChars, maxLines, opts)`:
- Перебирает валидные разбивки на ≤maxLines строк, каждая ≤maxChars.
- Скоринг кандидата: сильный штраф, если строка ЗАКАНЧИВАЕТСЯ glue-словом
  (предлог/союз/частица — курируемый список); штраф за дисбаланс длин строк;
  бонус, если разрыв совпадает с `opts.hintBreakAfter` (индекс слова —
  LLM-подсказка).
- Возвращает лучший кандидат; если ни один не влезает — fallback на текущий
  `_wrapWords`/best-effort пополам.
- Экспортировать. Юнит-тесты: висячий предлог уезжает вниз; баланс; лимит
  символов соблюдён; hint honored при валидности и игнор при невалидности;
  fallback на сверхдлинном слове.

Список glue-слов (курируемый, RU): предлоги (в, во, на, над, под, по, за, к,
ко, с, со, о, об, от, до, из, у, для, без, при, про, через, между, перед) +
союзы (и, а, но, да, или, либо, что, чтобы, как, когда, если, чем, то) +
частицы (не, ни, же, бы, ли, уж).

**T1.2 — Роутинг wrapCueLines (shared).**
Врезать в `buildKaraokeCues` (финальный `cueText`), `applyProofread`
(вместо пересборки по старым строкам — вызвать wrapCueLines с hint), и
`rebuildCueText` (правки из модалки). Тесты этих функций обновить.

**T1.3 — LLM-подсказка границы (panel.js).**
Расширить промпт `reelsProofread`: для титров, требующих 2 строки, модель
может вернуть текст с `\n` на естественной границе фразы. `applyProofread`
переводит позицию `\n` в `hintBreakAfter` и отдаёт в wrapCueLines. Guard
по-прежнему сравнивает плоский текст (число/порядок слов). Промпт остаётся
«только опечатки/пунктуация» для слов + необязательная подсказка переноса.

**T1.4 — Live-валидация Части 1** через CDP на реальной секвенции: прогнать
рилс, проверить, что в титрах нет висячих предлогов и строки сбалансированы.

## Часть 2 — Панель «Откатить»: обрезаются длинные имена секвенций

### Проблема
Кнопки попапа наследуют `white-space: nowrap` от `.action-bar button`
(index2.html:47); у `.undo-popover` жёсткий `max-width: 320px` (:118) без
переноса/эллипсиса → длинное имя секвенции клипается, непонятно куда жать.

### Задача (T2 — CSS, index2.html)
- `.undo-popover button`: `white-space: normal; word-break: break-word;`
  разумный `line-height`, `min-height`.
- `.undo-popover`: поднять `max-width` (или снять), задать комфортную ширину.
- `.u-when` (таймкод): `white-space: nowrap`, чтобы не разрывался.
Проверка визуально в CEP (Windows) + логика без изменений.

## Часть 3 — Тултипы не видны на macOS

### Проблема
Все подсказки — нативный `title=""`. В CEP WebKit на macOS нативные
title-тултипы не рендерятся. JS-фолбэка нет.

### Задача (T3 — panel.js + CSS)
Лёгкий кастомный тултип:
- Делегирование `mouseover`/`mouseout` на документе; для элемента с `title`
  (или `data-tt`) — сташить `title` в `data-tt` и убрать `title` (чтобы на
  Windows не было двойного нативного), показать позиционированный div;
  на `mouseout` — вернуть `title`.
- Позиционирование у курсора/элемента, clamp к границам панели, задержка ~400мс.
- CSS `.cc-tooltip` (тёмный, как остальной UI). Работает на обеих ОС.
- Проверка: тултипы появляются на Windows (и на macOS у пользователя).
