# «Рилс» — объединение «Вертикаль» + «Субтитры» в один пайплайн

Дата: 2026-07-17. Статус: дизайн утверждён пользователем.

## Цель

Одна карточка «Рилс» на вкладке Инструменты вместо карточек «Вертикаль 9:16» и
«Субтитры»: рефрейм секвенции в вертикаль/квадрат с vision-автопозиционированием
кадра + анимированные караоке-субтитры фирменным шрифтом, отрендеренные в
прозрачный оверлей поверх новой секвенции.

## Утверждённые решения

| Параметр | Решение |
|---|---|
| Форматы | 9:16 (1080×1920) и 1:1 (1080×1080), выпадашка |
| Позиционирование | Vision-модель Cloud.ru по кадру из середины каждого уникального источника; ручное поле смещений остаётся и **побеждает** vision |
| Шрифты | Пресеты: `SB Sans Text Semibold`, `SB Sans Display SemiBold` (установлены в системе, libass берёт системные) |
| Корректура LLM | Только орфография/пунктуация, без искажения содержимого; guard по словам — если модель переписала текст, кью остаётся исходным |
| Точки | Убирается только точка в самом конце титра (последний символ кью); `?` `!` `…` остаются; точки в середине титра остаются. Детерминированный код, НЕ LLM |
| Анимация | `нет` / `цвет слова` / `плашка под словом` — параметр |
| Цвета | Цвет текста (дефолт белый) и цвет подсветки/плашки (дефолт #21A038) — настраиваемые color-поля |
| Старые карточки | Обе удаляются из UI; host-функции остаются (переиспользуются) |
| Субтитры | Всегда рендер-оверлей (ffmpeg+libass, ASS), даже при «Анимация: нет» — иначе шрифт не применить. Нативный caption-трек в «Рилс» не используется |

## Отброшенные альтернативы

- **Нативный caption-трек для статичного режима** — API Premiere не задаёт шрифт
  caption-трека и не умеет пословную анимацию.
- **Рендер финального ролика целиком вне Premiere** — теряется монтаж в
  таймлайне и ручная доводка.
- **Пословные SRT-чанки** (слово = отдельный титр) — дёргается, шрифт всё равно
  дефолтный.

## Архитектура

### Поток данных

```
UI «Рилс» (формат, шрифт, анимация, цвета, ручные смещения)
 ↓ toolsRunReels() (panel.js)
 1. Транскрипт обязателен: ContextStore.findTranscriptEntry → entry.segments
    (нет → понятная ошибка «сначала транскрибируйте»)
 2. getVerticalReframeSources (host, есть) + отдача in/out клипов
 3. Vision-смещения (новое):
    - для каждого уникального mediaPath: ffmpeg -ss <середина> → jpeg ~768px
      → base64 → vision-модель Cloud.ru → bbox лица/спикера → offsetPct
    - ручные смещения по имени (parseVerticalOffsets, есть) перекрывают vision
    - nest / фейл vision → центр + предупреждение в отчёте
 4. planVerticalReframe (есть; targetW/H параметризованы) →
    applyVerticalReframe (host, есть) → секвенция «имя — Рилс 9x16|1x1»
 5. Караоке-кьюи (новое): титры с пословными таймкодами
    (Whisper word-level из кэша; fallback — линейная интерполяция),
    ~20 симв./строка, 2 строки (короче, чем 42 у старых сабов — вертикаль)
 6. LLM-корректура (батчи, строгий JSON, chat-модель) + guard;
    затем детерминированное правило точек
 7. ASS-генератор (чистая функция): стили из настроек; «цвет слова» —
    караоке-теги \k; «плашка» — BorderStyle=3 (BackColour) пословно
 8. ffmpeg-рендер: прозрачный оверлей ProRes 4444 (yuva444p10le), fps
    секвенции, длительность = конец последнего титра/секвенции.
    Проверка наличия фильтра ass в сборке ffmpeg — иначе стоп с инструкцией
 9. Импорт .mov + insertOverlayOnTopTrack (host, НОВОЕ): добавить верхнюю
    видеодорожку, положить клип с 0. Host-валидация, bump версии
 10. Отчёт: имя секвенции, смещения (vision/ручные/центр), число титров,
     число правок LLM
```

### Новые модули / функции

| Единица | Где | Ответственность |
|---|---|---|
| `reels-pipeline.js` | client/shared | Чистая логика: караоке-кьюи из сегментов, правило точек, guard корректуры, offsetPct из bbox, генерация ASS, план батчей корректуры |
| `toolsRunReels()` | client/unified/panel.js | Оркестрация: host-вызовы, vision, ffmpeg, LLM, статусы карточки |
| извлечение кадра | client/shared (reels-pipeline или audio-preprocess) | ffmpeg -ss mid -frames:v 1 → jpeg |
| `insertOverlayOnTopTrack` | host/premiere.jsx | Новая дорожка сверху + вставка клипа с 0, честные ok:false |
| UI-карточка «Рилс» | index2.html | Формат/шрифт/анимация/цвета/смещения/кнопка |

### Vision-контракт

Запрос: jpeg кадра (даунскейл до ~768px по большей стороне), промпт: «найди
главного человека/лицо в кадре, верни JSON {cx: 0..1} — горизонтальный центр».
`offsetPct = (cx − 0.5) × 100`, кламп ±50. Модель — vision-модель из настроек
(Cloud.ru; Qwen3.5-397B / MiniMax-M3). Ошибка/таймаут/невалидный JSON → центр.

### Корректура-контракт

Вход: массив `{i, text}` (текст кью без переносов), батчи по ~40. Промпт:
исправь орфографию и пунктуацию, НЕ меняй слова и порядок. Выход: строгий JSON
`[{i, text}]`. Guard: нормализованные слова (lowercase, без пунктуации) до и
после должны совпадать — иначе кью не трогаем, счётчик «отклонено guard'ом».
После корректуры код убирает финальную точку кью (правило точек).

## Обработка ошибок

- Нет транскрипта → стоп, «сначала транскрибируйте».
- Нет ffmpeg / нет фильтра ass / нет шрифта в C:\Windows\Fonts (или
  ~/Library/Fonts) → стоп до каких-либо изменений в проекте, инструкция.
- Vision недоступна → все клипы в центр, предупреждение, пайплайн идёт дальше.
- LLM-корректура упала → исходные титры, предупреждение.
- applyVerticalReframe/insertOverlay частично → честный ok:false (стандарт
  плагина), отчёт что успело создаться.

## Тесты

- Unit (TDD, vm-загрузка как у остальных shared-модулей): караоке-разбиение
  (word-level и fallback), правило точек (все случаи: `.`, `?`, `!`, `…`,
  точка в середине), guard корректуры (идентичные слова / искажение),
  offsetPct из bbox (центр/края/кламп), ASS-генерация (стили, \k-тайминги,
  плашка, цвета ARGB-формат ASS `&HAABBGGRR`), план батчей.
- Vision/ffmpeg/LLM/host — инжектируемые зависимости, в тестах моки.
- Live: CDP-прогон на реальной секвенции с транскриптом, визуальная проверка
  в Premiere.

## Чистка

- index2.html: карточки `card-vertical`, `card-subtitles` → одна `card-reels`.
- panel.js: `toolsRunVertical`, `toolsRunSubtitles` заменяются `toolsRunReels`
  (ветки в `toolsRunTool`).
- Host: `getVerticalReframeSources`, `applyVerticalReframe` переиспользуются;
  `importSrtAsCaptions` остаётся в host (не ломаем возможные вызовы), UI на
  него больше не ссылается.
