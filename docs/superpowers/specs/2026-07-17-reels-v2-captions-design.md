# Рилс v2: captions-слой, редактор титров, честный синхрон караоке

Дата: 2026-07-17. Статус: дизайн утверждён пользователем (вариант «C» — оба слоя;
источник правды правок — модалка панели, вариант «A»).

Надстройка над Рилс v1 (`2026-07-17-reels-design.md`). Мотивация: (1) караоке-анимация
не попадает в слова; (2) текст запечён в .mov и нередактируем.

## Ограничения платформы (проверено live 17.07.2026)

- **Cloud.ru Whisper НЕ отдаёт пословные таймкоды.** `timestamp_granularities[]=word`
  на живом аудио: 200 OK, но `words` нет ни в сегментах, ни на верхнем уровне.
  Единственная ASR-модель из 97 (`openai/whisper-large-v3`). Слова придётся
  выравнивать самим — детерминированно.
- **Caption-дорожку нельзя ни прочитать, ни удалить из ExtendScript.**
  `createCaptionTrack` только создаёт новую. Отсюда: правки captions в Premiere
  плагин не видит; при обновлении captions старую дорожку удаляет пользователь.
- **MOGRT/captions не принимают пословный тайминг** — караоке возможно только
  рендер-оверлеем (ASS + libass → ProRes 4444), как в v1.

## 1. Синхрон караоке — детерминированное выравнивание слов

Чистая функция в `client/shared/reels-pipeline.js` (замена линейной раздачи
в `buildKaraokeCues`):

1. **Взвешивание по длине слова**: интервал сегмента делится пропорционально
   числу символов слова (не поровну).
2. **Вычитание пауз**: если у entry есть `audioAnalysis.silences`
   (ffmpeg silencedetect, sequence-time, уже считается для «Тишин») — из
   интервала сегмента вычитаются пересекающие его тишины **≥0.3с**, слова
   раскладываются char-weighted только по озвученным подынтервалам
   (в хронологическом порядке, пропорционально длительности подынтервалов).
3. **Деградация**: нет audioAnalysis → чистый char-weighted по всему сегменту.
   Нативные `seg.words` (если когда-нибудь появятся) — приоритет, как в v1.

Тайминги кью (границы титров) не меняются — только распределение слов внутри.

## 2. Captions-слой — всегда, редактируемый нативно

- `toolsRunReels()` после сборки кью **всегда** генерирует .srt из финальных кью
  (после корректуры LLM и зачистки конечных точек) и импортирует caption-дорожкой
  в рилс-секвенцию через существующий `importSrtAsCaptions`
  (с `expectedSequenceName` = имя рилс-секвенции). `importSrtAsCaptions`
  работает по активной секвенции, поэтому порядок: рефрейм → оверлей
  (`importAndOverlayOnTop` активирует рилс-секвенцию) → captions.
  Для анимации «нет» оверлея не будет — нужен host-хелпер
  `activateSequenceByName(name)` (или флаг activateOnly), который активирует
  рилс-секвенцию перед импортом captions.
- Генерация SRT — чистая функция `buildSrt(cues)` в reels-pipeline
  (нумерация с 1, время `HH:MM:SS,mmm`, пустая строка между блоками).
- **Анимация «нет» → только captions, БЕЗ .mov**: не рендерим и не вставляем
  оверлей вообще. Полностью нативный, редактируемый результат.
- **Анимация «цвет слова»/«плашка» → captions + караоке-.mov.** В отчёте
  предупреждение: при включённом CC в Program-мониторе текст задвоится —
  выключить CC или удалить caption-дорожку.

## 3. Кью — источник правды в ContextStore

После сборки рилса кью сохраняются в ContextStore (тот же PID-механизм, что
транскрипт-кэш) по **имени рилс-секвенции**:

```
{ cues: [{startSec, endSec, text, words:[{w,s,e}]}],
  settings: {format, anim, font, textColor, hlColor, fps, w, h},
  paths: {srt, ass, mov}, sourceSequenceName, createdAt }
```

Переживает перезапуск панели. Обновляется при каждой пересборке и каждой правке
в модалке.

## 4. Модалка «Править титры»

Кнопка на карточке «Рилс» (рядом с «Собрать рилс»), активна когда для активной
(или производной рилс-) секвенции есть сохранённые кью. Паттерн UI — модалка
«Читать транскрипт».

- Список кью: таймкод (нередактируемый) + текстовое поле. Правка текста
  пересобирает `words` кью char-weighted внутри его [startSec, endSec]
  (с вычитанием тишин, как в §1). Границы кью не двигаются.
- **Без LLM-guard**: правки монтажёра авторитетны, применяются как есть.
- Кнопка **«Обновить captions»**: новый .srt → `importSrtAsCaptions` → новая
  caption-дорожка. В отчёте: «старую caption-дорожку удалите вручную»
  (API удаления нет — честное ограничение, не скрываем).
- Кнопка **«Перерендерить караоке»** (видна только если anim ≠ «нет»):
  ASS → ffmpeg → замена оверлея **на той же дорожке**. Доработка host
  `importAndOverlayOnTop`: параметр `replaceTopOverlay: true` — вместо
  добавления новой дорожки удалить/перезаписать клип на верхней видеодорожке
  (`overwriteClip` новым импортом @0). Число дорожек не растёт.

## 5. Тестирование

- **Юнит (TDD, tests/reels-pipeline.test.mjs)**: char-weighted распределение;
  вычитание тишин (пауза в середине сегмента; тишина < 0.3с игнорируется;
  тишина, накрывающая весь сегмент → деградация); `buildSrt` (формат времени,
  нумерация, CRLF/LF); пересборка words после правки текста кью; сохранение
  границ кью.
- **Live CDP**: полный цикл — сборка рилса (captions появились, караоке в
  синхроне на слух/по кадрам) → правка текста в модалке → «Обновить captions»
  (новая дорожка с новым текстом) → «Перерендерить караоке» (тот же трек,
  новый .mov, число дорожек не выросло) → anim «нет» (только captions, .mov
  не создан).

## Вне скоупа

- Round-trip правок из нативного окна Text Premiere (экспорт SRT → импорт) —
  добавим, если модалки окажется мало.
- Стилизация caption-дорожки (шрифт/цвет captions) — руками в Premiere.
- Удаление устаревших caption-дорожек — невозможно из ExtendScript.
