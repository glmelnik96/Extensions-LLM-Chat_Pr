## Где они объективно лучше (и почему)
1. **Vision** — судят монтаж по картинке, не только по тексту. Для нас, чья цель — «монтаж по
   смыслам», это прямой пробел: смысл часто в кадре, а не в словах.
2. **Устойчивость длинных операций** — WS + heartbeat + reconnect. У нас длинные вызовы —
   fire-and-forget через CDP-костыль, в проде — таймауты клиента.
3. **UX ошибок** — доменные коды + человеко-читаемые инструкции. У нас generic.
4. **Ширина фич** — джамп-каты, диаризация, авто-вертикаль, b-roll, цветокор, смарт-субтитры.
5. **Прозрачность стоимости** — контекст-поповер (% окна, system vs actions).

## Где мы лучше (сохранить как moat)
- Приватность (ничего не уходит в облако кроме LLM/Whisper-вызовов на Cloud.ru).
- Мгновенный host (код на диске, не round-trip на каждый tool-call).
- Самовалидация деструктивных host-операций (reject, не clamp) — у них этого нет.
- Заточка под RU-контур (Cloud.ru тарифы, ₽-счётчик).

## Принципы приоритизации
- **Сначала надёжность** (Волна 1): новые фичи на дырявом фундаменте усилят боль, не ценность.
- **Мерить качеством ролика**, не «код работает» — каждую фичу e2e на реальном мультикам-материале
  (1_SYNCED и др.), с ground-truth.
- **Только сведённый вход** для монтажа по смыслам (мультикам → nest/single-cam).
- **Не жертвовать moat**: любую «облачную» фичу конкурента реализуем локально/на Cloud.ru.
