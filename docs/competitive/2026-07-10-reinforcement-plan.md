> (Cloud.ru + локальный host + ffmpeg).

## Где они объективно лучше (и почему)
1. **Vision** — судят монтаж по картинке, не только по тексту. Для нас, чья цель — «монтаж по
   смыслам», это прямой пробел: смысл часто в кадре, а не в словах.
2. **Устойчивость длинных операций** — WS + heartbeat + reconnect. У нас длинные вызовы —
   fire-and-forget через CDP-костыль, в проде — таймауты клиента.
3. **UX ошибок** — доменные коды + человеко-читаемые инструкции. У нас generic.
4. **Ширина фич** — джамп-каты, диаризация, авто-вертикаль, b-roll, цветокор, смарт-субтитры.
5. **Прозрачность стоимости** — контекст-поповер (% окна, system vs actions).

## Где мы лучше (сохранить как moat)
- Приватность (ничего не уходит в облако кроме LLM/Whisper-вызовов на Cloud.ru).
- Мгновенный host (код на диске, не round-trip на каждый tool-call).
- Самовалидация деструктивных host-операций (reject, не clamp) — у них этого нет.
- Заточка под RU-контур (Cloud.ru тарифы, ₽-счётчик).

## Дорожная карта — 3 волны

### 🌊 Волна 1 — Надёжность (чиним свои дыры; блокер перед новыми фичами)
Приоритет по подтверждённому/дешёвому:
1. **429 `Retry-After`** ✅ **СДЕЛАНО 10.07** — parseRetryAfterMs + wait=max(backoff, header, cap 60с).
2. **Пересечения интервалов** ✅ **СДЕЛАНО 10.07** — JS-валидатор уже ловил (P1-F); host
   applyTranscriptCuts теперь отклоняет весь план ДО beginUndoGroup. Live-подтверждено.
   🆕 Бонус: закрыт live-обнаруженный P0 «24ч-wrap» — out-of-bounds время резало реальный
   контент; `_seqEndSec`-гард в applyTranscriptCuts + applyTimecodeEdits (см. аудит, статус-секция).
3. **Пустые Whisper-segments** ✅ **СДЕЛАНО 10.07** — assertNonEmptyTranscript на выходе runFromPrep.
4. **Cleanup temp-файлов при abort** транскрипции (регистр очистки в abort-handler).
5. **`_pendingProposal` / смена секвенции** — mutex + повторная `assertSequenceMatch` перед apply;
   тест на гонку.
6. **Утечка `_transcriptCheckpoints`** — TTL/лимит/чистка при cancel.
7. **NaN/bounds в маркерах и trackIndex** ✅ **СДЕЛАНО 10.07** — negative timeSec reject в host;
   NaN-гарды в интервалах; trackIndex-пункты аудита оказались stale (уже безопасны).
8. **Тесты** на все дыры из аудита §6 (пустая секвенция, пересечения, abort-SSE, temp-cleanup, гонка).
> Каждый пункт — сначала грепом подтвердить факт (см. VERIFY-метки в аудите), потом
> failing-тест → фикс → e2e через CDP на реальной секвенции.

### 🌊 Волна 2 — UX-паритет (дёшево, большой эффект)
1. **Человеко-читаемые ошибки** — перенять каталог сетевых/host-кодов (teardown §5.1, §2). Доменные
   коды host → дружелюбный алерт с инструкцией.
2. **Контекст/токен-поповер** — расширить usage-badge: «% окна контекста», разбивка system+tools vs
   actions, подсказка «начни новый чат когда большой». Данные уже есть в UsageMeter.
3. **Стек чекпоинтов** (rewind-точки) вместо одного undo — мультиоткат для серии правок.
4. **Пресеты экспорта `.epr`** под задачу (моно/low-res для транскрипции) — быстрее, меньше 25MB-лимит.

### 🌊 Волна 3 — Новые возможности (по ценности для «монтажа по смыслам»)
1. **⭐ Vision по кадрам** (наибольший стратегический буст): экспорт кадров ключевых сегментов →
   downscale 768 JPEG → мультимодальная модель Cloud.ru. Монтаж/маркеры/хуки судить по картинке+тексту.
   Проверить, какие Cloud.ru модели мультимодальны (см. project_model_benchmark память).
2. **Джамп-каты по тишине** — silence-detect (RMS+порог+padding) как инструмент; переиспользовать
   существующую механику удаления интервалов.
3. **Диаризация мультикама по спикерам** — бьёт прямо в нашу мультикам-боль (свести подкаст по
   говорящим). Синергия с уже решённой nest-транскрипцией.
4. **Смарт-субтитры** через MOGRT-шаблон (стилизованные, а не SRT).
5. **Авто-вертикаль/рефрейм** для шортсов.
6. **Устойчивость длинных операций** — оценить локальный аналог WS-heartbeat для прод-режима
   (без CDP-костыля): держать соединение/поллинг статуса長ых вызовов Cloud.ru.

## Принципы приоритизации
- **Сначала надёжность** (Волна 1): новые фичи на дырявом фундаменте усилят боль, не ценность.
- **Мерить качеством ролика**, не «код работает» — каждую фичу e2e на реальном мультикам-материале
  (1_SYNCED и др.), с ground-truth.
- **Только сведённый вход** для монтажа по смыслам (мультикам → nest/single-cam).
- **Не жертвовать moat**: любую «облачную» фичу конкурента реализуем локально/на Cloud.ru.

## Ближайший шаг (предложение)
Начать с Волны 1, п.1–3 (429 Retry-After + валидатор пересечений + пустые Whisper-segments) —

failing-тесты. Ждём отмашки на выбор пунктов.
